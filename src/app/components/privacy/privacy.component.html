🛡️ Sanctum of Shadows – Privacy & Security

1. Minimalist data rite

Only product‑centric attributes (Sigil, Name, Realm, Quantity, Price, Supplier, Aura) are stored. Absolutely no personal data, location, or device identifiers are collected or tracked.

2. Ephemeral storage covenant

This is a client‑side demo using in‑memory storage. In production, implement encrypted storage (iOS Keychain / Android Keystore) and avoid storing sensitive supplier contracts or cost prices in plain text.

3. Warding against curses (injection)

All inputs are validated: unique Sigil, numeric ranges, and HTML escaping to prevent XSS. Every user entry is sanitized before display.

4. Least privilege covenant

The application requests zero device permissions (camera, location, contacts). Mobile apps should always follow this principle, requesting only essential permissions with clear user explanation.

5. Veiled transmission (network)

Although this demo is front‑only, any production mobile app must enforce HTTPS/TLS 1.3, implement certificate pinning, and avoid sensitive data in URL parameters.

6. Transparent rituals

Delete operations require explicit confirmation; edit actions provide cancel options. Clear feedback messages appear after every mutation. Compliant with GDPR spirit.
